# Chatbot 對話 / 功能設計說明

本檔案概要描述 Line Bot（吃土觀測站）的對話流程、功能設計與使用情境，以符合作業繳交規範。

---

## 1. 系統角色與目標

- 角色：LINE 記帳小精靈
- 主要任務：協助使用者以自然語言記錄支出/收入、查詢統計與管理既有紀錄（更正 / 刪除）
- 特色：整合 OpenAI 語意解析、MongoDB 資料儲存、LINE Flex UI 呈現雙向統計

---

## 2. 主要互動流程

### 2.1 記帳
| 使用者訊息 | 系統行為 |
| --- | --- |
| `午餐 120` | 解析為「支出 / 餐點 / 午餐 / $120」，寫入 DB，回覆確認訊息 |
| `收入 50000`、`薪水 30000` | 解析為收入紀錄，寫入 DB，回覆帶情境的提示 |
| `星巴克 65`、`下午茶花了 90` | 以 LLM + fallback 規則判斷類別並記錄 |

### 2.2 查詢
| 指令 / Rich Menu | 說明 |
| --- | --- |
| `查詢統計`、Rich Menu「本月統計」 | 同步查詢支出/收入、計算淨收入並推送 Flex 圖卡 |
| `上個月花多少`、`這半年花多少`、`今年花多少` | 指定範圍的統計摘要 + Flex 圖表 |
| `餐點細項`、`11/25 花費細項`、`收入細項` | 檢索資料庫並列出細項清單 |
| `今天的記錄` | 列出當日所有支出 |

### 2.3 更正 / 刪除
| 指令 | 系統行為 |
| --- | --- |
| `第一筆要改成飲品`、`更正今天午餐為120` | 使用 LLM 解析候選紀錄 → 要求使用者確認 → 更新資料 |
| `刪除第3筆`、`刪除昨天的餐點` | 解析索引 / 關鍵字 / 日期，刪除對應紀錄並回覆結果 |

---

## 3. Rich Menu 設計
| 區域 | 功能 |
| --- | --- |
| 使用說明 | 回覆 `getHelpMessage()`，列出所有語句示例 |
| 每月開銷 | 彈出年份 / 月份選單（Quick Reply） |
| 本月統計 | 呼叫 `/api/line` 的 postback，回覆支出/收入/淨收入文字 + Flex |
| 今日記錄 | 查詢今天的所有支出列表 |

---

## 4. Flex UI
1. **統計摘要 Flex**：在文字摘要後附上支出 / 收入占比條圖（或改為圓餅圖）
2. **空資料處理**：若支出/收入無資料，改用文字提示並降級為純文字

---

## 5. 錯誤處理 / 降級策略
| 情境 | 處理方式 |
| --- | --- |
| OpenAI API 失敗 | 使用 `fallbackParsedMessage` 做基本解析 |
| MongoDB / Line API 失敗 | 回覆友善訊息，並在伺服器端記錄 |
| 更正/刪除無結果 | 引導使用者輸入序號或更明確描述 |

---

## 6. 管理後台
- 位置：`/admin`
- 功能：對話紀錄、月報表、即時 SSE、統計檢視
